## 배운 것
- Gemini 대화 API는 role이 반드시 `user → model → user → model` 순으로 교차해야 함. 연속 같은 role이나 `model`로 시작하면 API 에러
- React StrictMode는 개발 환경에서 `useEffect`를 의도적으로 두 번 실행함 → ref로 최초 1회 실행을 보장해야 함
- `systemInstruction`을 활용하면 `contents`의 role 순서를 건드리지 않고 평가 기준을 주입할 수 있음
- Vercel Hobby 플랜은 서버리스 함수 타임아웃 10초 → `thinkingBudget: 0`으로 응답 시간 단축이 현실적인 대응

## 느낀 것
- API 설계 시 role 교차 규칙처럼 "당연해 보이는" 제약 조건을 먼저 파악해두는 게 중요함. 나중에 디버깅하기 훨씬 어려워짐
- 에러 핸들링을 나중으로 미루면 사용자 경험이 크게 나빠짐. 처음부터 try-catch + 안내 메시지를 함께 설계하는 습관 필요
